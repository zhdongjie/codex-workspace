# name
code-review

# goal
根据项目的架构规范和代码风格要求，对指定的代码文件或模块进行严格审查

# steps
1. 使用 filesystem 获取需要审查的目标代码内容
2. 根据目标代码的技术栈（Vue3 或 Spring Boot），静默读取对应的 `.codex/rules/.../code-style.md` 规范文件
3. 逐行比对代码，检查是否存在违背“架构分层”、“命名约定”、“零魔法值”、“异常与日志规范”等红线原则的情况
4. 检查潜在的逻辑漏洞、边界条件（如空指针、数组越界）和性能隐患
5. 输出格式化的 Review 报告，列出“必须修改 (Blocker)”和“建议优化 (Minor)”的改进项

# constraints
- 除非人类明确授权，否则**只输出审查意见，绝对不允许直接修改源代码**
- 审查意见必须具体到行号，并给出符合规范的代码修改示例
- 语气需保持客观、专业，直指代码问题
