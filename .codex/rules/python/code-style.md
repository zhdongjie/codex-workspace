# Python Code Style

## 1. 核心语法与类型系统
- **强制类型提示 (Type Hints)**：充分利用 Python 3.13 等现代版本的特性，所有函数签名、类的方法必须包含参数类型和返回值类型注解。
- **PEP 8 规范**：严格遵守 PEP 8 命名标准（变量和函数使用 `snake_case`，类名使用 `PascalCase`，常量使用 `UPPER_SNAKE_CASE`）。

## 2. 异步与性能
- **异步编程优先**：在处理网络请求、数据库读写或外部模型调用等 I/O 密集型任务时，必须使用 `asyncio` 和 `async/await` 语法。
- **避免阻塞**：严禁在异步函数中调用同步的阻塞 I/O 操作。

## 3. API 与架构规范
- **数据验证**：如果开发 API（如使用 FastAPI），强制使用 Pydantic 模型进行请求体和响应体的数据校验。
- **模块化设计**：复杂的逻辑（如 RAG 检索链路、数据处理 Pipeline）必须拆分为独立的模块或类，避免在单个脚本中堆砌过程式代码。

## 4. 健壮性与环境
- **明确依赖**：引入新库必须更新对应的依赖文件，并在虚拟环境中运行。
- **异常捕获**：禁止使用裸露的 `except:`。必须捕获具体的异常类型，并使用 `logging` 模块记录完整的错误堆栈，生产代码中禁止遗留 `print()`。
